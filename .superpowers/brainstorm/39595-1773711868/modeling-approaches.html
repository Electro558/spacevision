3D Modeling Framework: Choose Your Path

Each approach adds TinkerCAD-like capabilities to your existing Three.js + R3F stack

A

react-three-csg + manifold-3d

TinkerCAD-style boolean modeling

Adds CSG booleans (union/subtract/intersect) directly into your existing R3F scene as JSX components. Manifold WASM validates geometry on export.

Pros

  • Lightest weight (~200KB added)
  • Native R3F/Three.js integration
  • Fastest to ship (1-2 weeks)
  • Declarative JSX API
  • Your existing code stays intact

Cons

  • Mesh booleans only (no NURBS/fillets)
  • Must build snap/align UI yourself
  • Floating-point edge cases
  • No STEP/IGES file support
B

OpenCascade.js (WASM) + Three.js

Industrial-grade CAD kernel in the browser

Full OpenCascade Technology compiled to WASM. Real B-Rep modeling with fillets, chamfers, NURBS surfaces, parametric sketching, STEP/IGES export.

Pros

  • Professional CAD-grade geometry
  • Fillets, chamfers, lofts, sweeps
  • STEP/IGES/BREP file formats
  • Custom builds as small as ~2.4MB
  • 80-90% native speed

Cons

  • Steep learning curve (C++ API)
  • More complex integration (3-5 weeks)
  • Larger bundle size
  • Requires bridging to Three.js renderer
C

Hybrid: react-three-csg NOW + OpenCascade LATER

Ship fast, upgrade incrementally

Start with react-three-csg for immediate boolean ops + snap/align tools. Layer in OpenCascade.js later for fillets, parametric features, and pro file formats as needed.

Pros

  • Working booleans in days, not weeks
  • Incremental complexity
  • Best of both worlds over time
  • Minimal disruption to existing code
  • Can evaluate OCCT before committing

Cons

  • Two geometry systems to maintain
  • Migration overhead later
  • Architecture decisions needed upfront

What You Currently Have

Three.js + React Three Fiber + Drei with 10 primitive shapes, full transform gizmos, color/material editing, text-to-3D generation, STL export, undo/redo, and keyboard shortcuts. No CSG/boolean ops yet.
