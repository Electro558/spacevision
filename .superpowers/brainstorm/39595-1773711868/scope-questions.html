Scoping the TinkerCAD-style Modeling

A few quick questions to define what we're building

1. How should boolean operations work in the UI?

A

TinkerCAD-style: Drag → Overlap → Group

User places shapes, positions them overlapping, selects multiple shapes, then clicks "Group" (union) or marks a shape as a "Hole" (subtract). Simple and intuitive.

B

Explicit Boolean Toolbar

Select two shapes → click Union / Subtract / Intersect from a toolbar. More flexible (supports intersection), more traditional CAD workflow.

C

Both — TinkerCAD mode + Advanced mode

Default to TinkerCAD's Group/Hole paradigm for simplicity, with an "Advanced" toggle that exposes Union/Subtract/Intersect explicitly.

2. What snapping/alignment features matter most?

1

Grid Snapping

Objects snap to a configurable grid (1mm, 5mm, 10mm increments)

2

Snap-to-Face / Workplane

Place new objects directly on the surface of existing ones (TinkerCAD's killer feature)

3

Align & Distribute

Select multiple objects → align centers/edges, distribute evenly

4

Smart Rulers / Dimension Inputs

See exact distances between objects, type precise dimensions while dragging

3. Priority for first build?

A

Booleans First

Get CSG operations working ASAP, add snapping later

B

Full Package

Build booleans + snapping + alignment together as one feature drop
